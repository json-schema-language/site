Stop writing JSON boilerplate

JSON Schema Language was made by developers tired of repetitive type-checks and crashes due to malformed JSON.

Before

Lots of manual checking. Easy to get wrong, and not portable. 

// There's a couple of problems with this code.
//
// 1. Only one error returned at a time.
// 2. Errors are sometimes misleading; the same message is given if an email is
//    missing versus being of the wrong type.
// 3. This code is not portable. If I want some other service, in a different
//    language, to do this very same validation, I'll have to write and maintain
//    this logic twice.
app.post("/users", (req, res) => {
  const data = req.body

  if (typeof data !== "object") {
    res.status(400).send({ error: "body must be an object" })
  }

  if (typeof data.email !== "string") {
    res.status(400).send({ error: "email must be a string" })
    return
  }

  if (typeof data.isAdmin !== "boolean") {
    res.status(400).send({ error: "isAdmin must be a boolean" })
    return
  }

  if (!Array.isArray(data.favoriteNumbers)) {
    res.status(400).send({ error: "favoriteNumbers must be a boolean" })
    return
  }

  for (const number of data.favoriteNumbers) {
    if (typeof number !== "number") {
      res.status(400).send({ error: "elements of favoriteNumbers must be numbers" })
      return
    }
  }

  // Ok, the data is valid. Let's proceed.
  //
  // ...
});
After

All checking is automated. Logic is portable between languages.

This example uses the TypeScript implementation and the jsl-codegen code generation command-line tool. 

import { Validator } from "@json-schema-language/json-schema-language";

// If you're using TypeScript, a tool called jsl-codegen can generate types from
// a schema.
import User from "./user"

const validator = new Validator({
  properties: {
    email: { type: "string" },
    isAdmin: { type: "boolean" },
    favoriteNumbers: {
      elements: { type: "number" }
    }
  }
})

app.post("/users", (req, res) => {
  try {
    // Check if the data is valid. Throws if not.
    validator.validate(req.body)

    // Ok, the data is valid! Let's proceed.
    //
    // ...
  } catch (err) {
    // err.errors will be an array of errors. The format of these errors is
    // standardized, and is the same in every implementation of JSON Schema
    // Language.
    res.status(400).send({ error: err.errors })
  }
})
